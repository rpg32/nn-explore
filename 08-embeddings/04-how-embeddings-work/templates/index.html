From Standalone to End-to-End

2013 (Word2Vec): Train embeddings separately, then use them as input features for other models. A two-step process.

2018+ (GPT, BERT): The embedding matrix is the first layer of the transformer. It's trained end-to-end — the embeddings are optimized for the exact task the model performs. No separate step.

In GPT-2, the embedding is literally: token_id → row of a 50,257 × 768 matrix → 768 numbers. Those 768 numbers flow into the attention layers. The whole thing trains together.

Key insight: An embedding is just a lookup table (a matrix). One-hot × matrix = select a row. That row is the word's representation. In Word2Vec it's trained to predict context. In GPT it's trained to predict the next token. Same mechanic, different training signal.

Things to Try

  • Click each word in Section 1. Notice that the forward pass is: one-hot → select a row from W_embed → that's the embedding. The matrix multiply is just a fancy row selection.
  • Look at the embedding matrix. Row 0 = "cat"'s vector. Row 1 = "dog"'s vector. Each row is just 2 numbers — the coordinates in embedding space.
  • Click Train and step through snapshots. At epoch 0, all words are random. By epoch 30, "cat" and "dog" are near each other because both predict similar context words.
  • Look at the training steps. When center="cat" predicts context="sat", the gradient adjusts cat's row. When center="dog" also predicts context="sat", dog's row moves the same way. Convergence!
  • Read the real-world table. GPT-2's embedding matrix is 38.6M parameters — and it's the smallest part of the model. The transformer layers on top are where the real processing happens.
