Gradient Descent

The algorithm that makes neural networks learn — rolling downhill on the loss landscape

The loss landscape maps every possible weight setting to its loss value — think of it as terrain. Gradient descent finds the lowest point (best weights) by repeatedly stepping downhill.

wnew = wold - learning_rate × gradient

The gradient points uphill (steepest increase). We go the opposite direction. The learning rate controls how big each step is. Click the landscape to set a starting point, then step or run to watch the descent.

Things to Try

  • Click different starting points on the landscape. Does descent always reach the same minimum?
  • Simple Bowl + lr=0.1: Clean, smooth descent. Step through one at a time to see each gradient arrow.
  • Elongated Valley + lr=0.1: Watch it zigzag! The gradient points sideways across the narrow valley instead of along it. This is a classic problem.
  • Crank lr to 1.0 on any landscape. Watch it overshoot and bounce wildly — it may never converge!
  • Set lr to 0.001. Now it barely moves. Learning is agonizingly slow. Finding the sweet spot matters.
  • Saddle + Bowl: Start near the saddle region (around the center). The gradient is tiny there — descent almost stalls.
