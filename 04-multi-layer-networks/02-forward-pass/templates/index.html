The Forward Pass

Watch data flow through a network — one layer at a time, showing the actual matrix math

What Happens at Each Layer

The forward pass is the same operation repeated at each layer: take the previous output, multiply by a weight matrix, add a bias vector, apply an activation function. That's it — the entire computation of a neural network is this pipeline applied layer by layer.

input vector × weight matrix + bias → pre-activation (z) → activation → output

Things to Try

  • Start with "2 → 1" (simplest). Only one layer of math: 2 inputs multiplied by a 1×2 weight matrix + 1 bias → sigmoid. This is the single neuron from Module 01.
  • Step through "2 → 3 → 1". Watch how 2 inputs become 3 hidden values, then collapse to 1 output. Notice the weight matrix shapes change at each layer.
  • Move the input sliders. Then step through again. The same weights produce completely different intermediate values — inputs ripple through the whole network.
  • Click "Show All" to see every layer at once. Then look at how the vector sizes change: 2 → 4 → 3 → 1 means the data gets expanded, mixed, then compressed down to a single prediction.
  • Look for dead neurons. In the hidden layers, some neurons might have z < 0, so ReLU kills them to 0. Those neurons contribute nothing to the output. This is the "dying ReLU" problem.
  • Notice the matrix shapes. Layer going from 4 neurons to 3 neurons has a 3×4 weight matrix. The number of rows = output neurons, columns = input neurons. This is the key pattern.
