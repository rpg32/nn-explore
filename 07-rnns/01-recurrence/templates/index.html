Module 07 · Sequential Processing

Recurrent Neural Networks

Processing sequences one element at a time, carrying memory in a hidden state

An RNN processes a sequence one element at a time. At each step, it takes the current input AND the previous hidden state, combines them, and produces a new hidden state. The hidden state is the network's memory — a compressed summary of everything it has seen so far.

h[t] = tanh(W_h @ h[t-1] + W_x @ x[t] + b)
Same weights W_h and W_x reused at EVERY time step — weight sharing across time

Vanishing Gradient Through Time

Gradient strength flowing backward from the last step

Just like deep layers (Module 03.3), gradients weaken as they flow backward through many time steps. Step 30 has full gradient. Step 1's gradient has been multiplied through 29 tanh activations — nearly zero. This is why basic RNNs can't learn long-range dependencies. LSTMs fix this with a gradient highway.

Things to Try

  • Process "hello". Click each step. Watch the hidden state (the colored grid) change as each character is processed. After 'h', the memory encodes 'h'. After 'he', it encodes a mix of both. By 'hello', it carries information about the whole word.
  • Try "abcabcabc". Notice the hidden states become PERIODIC — the memory pattern at 'a' repeats every 3 characters because the input repeats. The RNN has discovered the pattern.
  • Try "the cat sat." More characters = more information compressed into the same-size hidden state. The network has to decide what to remember and what to forget.
  • Look at the gradient chart. This is Module 03.3 (vanishing gradients) happening across time instead of across layers. The network can barely update its weights based on early characters — it effectively "forgets" them.
  • The predictions are random because this network is untrained. In a trained RNN, after seeing "hell", it would predict "o" with high probability.
