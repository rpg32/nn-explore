Data Normalization

Why raw data breaks training and how scaling your inputs fixes it

Same moons dataset, but x1 is scaled up 100× to simulate real-world data where features have different units (e.g., house price in dollars vs number of rooms). The gradient for w1 becomes 100× bigger than w2.

Raw:           x1 ∈ [-150, 150],   x2 ∈ [-1.5, 1.5]   — gradients wildly imbalanced
Standardized: x = (x - mean) / std                — both features: mean=0, std=1
Min-Max:      x = (x - min) / (max - min)        — both features: range [0, 1]

Things to Try

  • Run with default scale ratio 100×. The raw network barely learns — its learning rate had to be cut 10,000× just to survive. Standardized and min-max both converge quickly.
  • Set scale ratio to 1× (equal scales). Now raw data works fine! The problem only appears when features have different magnitudes.
  • Increase to 1000×. Raw data becomes nearly impossible to train. Normalized methods are unaffected — that's the point.
  • Look at the raw data heatmap. The boundary is nearly vertical because the network can only "see" the dominant feature (x1). The small feature (x2) is noise to it.
  • This is a preprocessing step, not a training trick. You normalize ONCE before training. It costs nothing and almost always helps. There's no reason not to do it.
