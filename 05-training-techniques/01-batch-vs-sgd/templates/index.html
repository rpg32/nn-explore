Batch vs SGD

Same network, same data, three ways to feed gradients — watch the difference

Full Batch: gradient = avg over ALL 200 points → 1 update/epoch
Mini-Batch: gradient = avg over B points → N/B updates/epoch
Stochastic: gradient = 1 point → N updates/epoch

All three start from identical weights. Same data, same network [2→16→8→1 ReLU]. Only the batch size differs. An epoch = one full pass through all data. More updates per epoch ≠ better — each update is noisier.

Things to Try

  • Run with defaults (30 epochs). All three reach similar accuracy, but look at the loss curves — full batch is smooth, SGD is noisy, mini-batch is in between.
  • Set epochs=5. SGD and mini-batch often converge faster because they make more weight updates per epoch. Full batch only gets 5 updates total!
  • Increase learning rate to 1.0+. Full batch stays stable. SGD may diverge — the noisy gradients amplified by a large LR can blow up. Mini-batch handles it better than SGD.
  • Change mini-batch size. Size 4 acts like SGD (noisy). Size 128 acts like full batch (smooth). Size 16-32 is the sweet spot most practitioners use.
  • Why does everyone use mini-batch? Real datasets have millions of samples. Full batch is impossible — you can't fit it all in memory. SGD wastes GPU parallelism. Mini-batch (32-256) is the practical sweet spot.
