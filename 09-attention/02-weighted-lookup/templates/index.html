Module 09 · Attention

Attention as Weighted Lookup

Query, Key, Value — the mechanism that replaced the bottleneck

The Library Analogy

Attention works like searching a library:

Query (Q): "What am I looking for?" — Your search question.

Key (K): "What does each book's label say?" — Each source word advertises its content.

Value (V): "What's inside each book?" — The actual information to retrieve.

You compare your query against every key. Books with matching labels get high relevance scores. You then read a weighted blend of all the books, proportional to their relevance.

1. score[i] = Q · K[i] / √d    (dot product — how well does query match each key?)
2. weights = softmax(scores)    (normalize to probabilities that sum to 1)
3. output = Σ weight[i] × V[i]    (weighted blend of all values)

Q, K, V are computed from the word embeddings using three learned weight matrices — same matrix multiply as every other layer we've built. Nothing new mathematically.

Example Query

1
The Query — "What am I looking for?"
The query word's embedding is multiplied by W_Q to produce the query vector.
2
The Keys — "What does each word offer?"
Each source word's embedding is multiplied by W_K. These are the "labels" that the query searches against.
3
Scores — "How well does my query match each key?"
Dot product of query with each key, divided by √d for stability. Higher = better match.
4
Attention Weights — "How much to attend to each word?"
Softmax converts scores to probabilities (sum to 1). This is the core output of attention — which words matter most.
5
Output — Weighted blend of all values
Multiply each value by its attention weight and sum. The result is a custom summary focused on what the query needed.

The key: Instead of one fixed summary (the bottleneck), each query gets a DIFFERENT weighted blend. When translating "le chat", the decoder attends to "cat". When translating "sur le", it attends to "on the". Each output step gets exactly the information it needs.

Things to Try

  • "Looking for an animal" (query: cat). The attention weight is highest on "dog" — it's the most similar key. "the" and "on" get low weights. The network learns to match semantic categories.
  • "Looking for a verb" (query: ran). Highest weight goes to "sat" (both verbs). Nouns and function words are mostly ignored.
  • Compare the scores vs weights. Scores are raw dot products (can be any number). Softmax squashes them into probabilities. A small score difference becomes amplified — the winner takes most of the weight.
  • Notice: this is all just matrix multiplies and a softmax. Q = W_Q × embedding. K = W_K × embedding. Score = Q · K. Nothing beyond what we've been doing since Module 01. The power comes from WHAT it computes, not HOW.
