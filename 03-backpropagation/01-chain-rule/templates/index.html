The Chain Rule

How gradients flow backward through a computation graph

In Module 02.4, computing gradients was easy — the neuron was a single step. But deep networks have chains of operations. The chain rule says: the gradient through a chain is the product of all local gradients.

x → [× w] → a → [+ b] → z → [sigmoid] → pred → [loss] → L

Forward pass flows left to right, computing values. Backward pass flows right to left, multiplying local gradients. Toggle between views below to see each direction.

Things to Try

  • Toggle "Forward Pass" first. See how x flows through each operation left to right, producing values at each node.
  • Toggle "Backward Pass". Now see gradients flow right to left. Each node multiplies the incoming gradient by its LOCAL derivative.
  • Toggle "Both" to see the full picture. Forward values on top, backward gradients on bottom.
  • Set x=0. Watch dL/dw become 0 — because da/dw = x = 0. If the input is zero, the weight gradient vanishes!
  • Set w=0. Watch dL/dx become 0 — the gradient can't flow through a zero weight.
  • Make z very large or very small (by adjusting w and x). Watch the sigmoid local gradient shrink toward 0. This is the vanishing gradient problem — previewed here, explored fully in Module 03.3.
  • Read the chain multiplication at the bottom. Every gradient is just the product of simple local derivatives. That's the entire idea behind backpropagation.
