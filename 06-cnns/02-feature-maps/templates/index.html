Feature Maps & Filters

A CNN applies MULTIPLE kernels simultaneously — each one detects a different feature in the same image

In Module 06.1, we applied one kernel at a time. But a real convolutional layer uses many kernels simultaneously — typically 32 or 64 in the first layer alone. Each kernel produces its own feature map, and together they capture a rich description of the input.

Below, 8 hand-crafted kernels are applied to the same image. Notice how each kernel lights up on different structures — horizontal edges, vertical edges, diagonals, corners. A CNN layer does exactly this, but with learned kernels.

🔑 Key Insight

In a real CNN, these kernels are LEARNED, not hand-designed. The network discovers through training which kernels are useful for the classification task. It starts with random kernels and backpropagation adjusts them — the network might learn edge detectors, texture detectors, or entirely novel patterns that no human would think to design.

Things to Try

  • Cross image: Compare the Horizontal Edge vs Vertical Edge maps — one finds the horizontal bar, the other finds the vertical bar. Together they describe the whole shape.
  • Digit "7": Horizontal Edge lights up on the top bar; Diagonal ↘ lights up on the angled stroke. A CNN combines these feature maps in later layers to recognize "this is a 7".
  • Circle: The Laplacian highlights the entire outline uniformly, while directional edge kernels only find edges aligned with their direction. That's why CNNs need multiple kernels!
  • Checkerboard: Every edge kernel goes wild — there are edges everywhere. But Blur smooths it all out. Notice how Sharpen makes the grid even crisper.
  • Letter "L": The Corner kernel responds where the vertical stroke meets the horizontal base. This is how CNNs detect junctions and intersections.
  • Click any feature map to see the kernel values that produced it. Notice how negative values (red) detect dark regions and positive values (green) detect bright regions.
