Convolution

A small filter slides over an image, computing weighted sums — detecting features everywhere

A convolution is just a weighted sum — the same operation a neuron does! But instead of connecting to ALL pixels, a small 3×3 kernel slides across the image, computing a weighted sum at each position. The output is a feature map showing where the kernel's pattern was detected.

Hover over the input image to see the kernel position. Hover over the feature map to see what was computed at that spot.

Things to Try

  • Horizontal Stripes + Vertical Edge kernel — nothing detected (no vertical edges in horizontal stripes). Now switch to Horizontal Edge kernel — lights up everywhere!
  • Circle + All Edges — the feature map shows the circle's outline. The interior and exterior are zero — only the BOUNDARY has signal.
  • Digit "7" + each edge kernel — horizontal edge finds the top bar, vertical edge finds the downstroke. This is how CNNs decompose shapes into parts.
  • Edit the kernel values. Set the center to 1, everything else to 0 — that's identity (output = input). Now set center to -1 — you get an inverted image.
  • Hover over the feature map. At each bright spot, the step detail shows exactly which 3×3 patch multiplied by which kernel values produced that output. It's just a weighted sum!
  • Notice the output is 26×26, not 28×28. The kernel can't center on edge pixels — it loses one pixel on each border. This is why real CNNs use "padding" to keep the size.
